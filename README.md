# AI News Digest — ежедневный дайджест ИИ-новостей из Telegram

Собирает вчерашние посты из открытых TG-каналов → суммаризирует через **Claude Code
в headless-режиме по вашей подписке Max** (без API-ключа и pay-as-you-go) → публикует
дайджест в ваш канал. Запускается по cron.

## Как это работает

```
cron (08:00) → digest.py
                 ├── Telethon (MTProto, ваш обычный TG-аккаунт) — читает каналы
                 ├── claude -p (Claude Code, авторизация по подписке Max) — саммари
                 └── Bot API (sendMessage) — постит в канал
```

## Установка (на любом Linux-сервере или домашней машине, всё в порядке и на WSL)

### 1. Python-зависимости

```bash
cd ai-digest
python3 -m venv venv && source venv/bin/activate
pip install telethon httpx python-dotenv
```

### 2. Claude Code + авторизация подпиской

```bash
npm install -g @anthropic-ai/claude-code
claude   # интерактивный вход: выбрать "Claude account with subscription", войти через браузер
```

После входа токен сохраняется локально, и `claude -p` работает от подписки.
Проверка: `echo "скажи ок" | claude -p` — должен ответить без запроса API-ключа.

Для сервера без браузера есть вариант `claude setup-token` — генерирует
долгоживущий OAuth-токен для скриптов (тоже от подписки). Либо авторизуйтесь
на локальной машине и скопируйте `~/.claude/` на сервер.

**Убедитесь, что в окружении/`.bashrc`/crontab НЕ задан `ANTHROPIC_API_KEY`** —
иначе headless-запуски пойдут в платный API, а не в подписку. Скрипт удаляет
переменную сам, но лучше, чтобы её не было вовсе.

### 3. Telegram-часть

1. **MTProto-доступ (чтение каналов):** зайдите на https://my.telegram.org/apps
   своим обычным аккаунтом, создайте приложение, получите `api_id` и `api_hash`.
2. **Бот (публикация):** создайте бота у @BotFather, получите токен,
   добавьте бота **администратором** в ваш канал-дайджест с правом публикации.
3. Скопируйте `.env.example` → `.env` и заполните.
4. Скопируйте `channels.txt.example` → `channels.txt` и впишите свои источники
   (дальше список можно менять командами бота `/add` и `/remove`).

### 4. Первый запуск

```bash
python digest.py --dry-run
```

При первом запуске Telethon спросит номер телефона и код подтверждения —
это одноразово, сессия сохранится в `digest_session.session`.
`--dry-run` выведет дайджест в консоль, не публикуя. Когда всё нравится:

```bash
python digest.py
```

### 5. Cron

```bash
crontab -e
```

```cron
# каждый день в 08:00 по времени сервера — дневной дайджест
0 8 * * * cd /home/USER/ai-digest && ./venv/bin/python digest.py >> digest.log 2>&1
# по воскресеньям в 21:00 — недельный обзор ключевых новостей
0 21 * * 0 cd /home/USER/ai-digest && ./venv/bin/python digest.py --weekly >> digest.log 2>&1
# каждый день в 13:00 — научный разбор статьи
0 13 * * * cd /home/USER/ai-digest && ./venv/bin/python paper.py >> digest.log 2>&1
```

Если время сервера — UTC, а дайджест нужен в 08:00 МСК, ставьте `0 5 * * *`,
либо задайте `CRON_TZ=Europe/Moscow` в начале crontab (поддерживается не везде).

Важно для cron: PATH в cron минимальный, поэтому если `claude` не находится —
укажите полный путь в `digest.py` (узнать: `which claude`) или добавьте
`PATH=/usr/local/bin:/usr/bin:/bin:...` в crontab.

## Настройка под себя

- **Промпт** — константа `PROMPT_TEMPLATE` в `digest.py`: категории, тон, объём.
- **Фильтры** — минимальная длина поста (80 символов), обрезка длинных постов
  (2500 символов), лимит 300 сообщений на канал в сутки.
- **Модель** — `CLAUDE_MODEL` в `.env` (`sonnet` — оптимально; `opus` — качественнее,
  но заметнее тратит лимит подписки).
- **Дедупликация между днями** — опубликованные выпуски сохраняются в `history/`, и при
  генерации модель видит последние из них, чтобы не повторять уже освещённые новости.
  Глубина просмотра — `DEDUP_LOOKBACK_DAYS` в `.env` (по умолчанию 2; `0` — выключить).
- **Недельный обзор** — `python digest.py --weekly` собирает обзор ключевых новостей недели
  из сохранённых в `history/` дневных дайджестов (до 7 последних) и публикует в тот же канал.
  В Docker запускается планировщиком по воскресеньям в 21:00 (`WEEKLY_AT`, `WEEKLY_DOW` в
  `docker-compose.yml`).

## Ежедневный научный разбор статьи

`paper.py` (в Docker — планировщик в 13:00) публикует детальный разбор одной свежей
научной статьи в стиле д.т.н.: строго, но понятно, с пояснением терминов, по схеме
идея → метод → как проверяли → результаты → применение, без смайликов и со ссылкой на arXiv.

Как выбирается статья: из постов каналов за последние `PAPER_LOOKBACK_DAYS` дней (по
умолчанию 3) вытаскиваются arXiv-ссылки; уже разобранные (см. `paper_history.txt`)
исключаются; если кандидатов несколько — модель выбирает самую значимую. Дальше
скачивается PDF, из него берётся текст (и метаданные с arXiv API), и `claude -p` пишет разбор.

```bash
python paper.py                  # разбор за последние дни
python paper.py --dry-run        # не постить, вывести в stdout (+ превью иллюстрации)
python paper.py --id 2401.12345  # принудительно разобрать конкретную статью
```

- **Иллюстрация (best-effort).** Из PDF извлекается крупнейшее растровое изображение
  (график/скрин) и публикуется как фото + текст. Векторные фигуры извлечь нельзя —
  тогда пост выходит текстовым.
- **Модель** — `PAPER_MODEL` в `.env` (по умолчанию как `CLAUDE_MODEL`); для более
  качественного разбора можно поставить `opus`.

## Живые вечерние посты

`vibe.py` раз в `VIBE_EVERY_DAYS` дней (по умолчанию 2) публикует лёгкий пост в
случайную минуту вечернего окна `VIBE_WINDOW_START`–`VIBE_WINDOW_END` (18:00–20:00).
Тип выбирается случайно:

- **joke** — короткая умная шутка про ИИ;
- **future** — размышление о том, куда движется ИИ и каким может быть будущее;
- **repo** — разбор популярного GitHub-репозитория про ИИ (ищется через GitHub
  Search API по AI-топикам, из топа модель выбирает действительно известный проект,
  уже показанные хранятся в `vibe_repos.txt`; для таких постов включено превью ссылки).

Интервал стережёт сам скрипт (`vibe_last.txt`), поэтому планировщик может дёргать
его ежедневно. Ручной запуск и отладка:

```bash
python vibe.py --dry-run                 # случайный тип, вывести в stdout
python vibe.py --type repo --dry-run     # принудительный тип
python vibe.py --type joke --force        # опубликовать сейчас, игнорируя интервал
```

Модель — `VIBE_MODEL` в `.env` (по умолчанию как `CLAUDE_MODEL`).

## Отчёт о трендах за 3 месяца

`trends.py` — разовый скрипт: собирает посты каналов за последние N месяцев
(по умолчанию 3) и сводит их в markdown-документ о ключевых трендах ИИ со ссылками
на источники. Из-за объёма работает по схеме map-reduce (батчи → выжимки → синтез).

```bash
docker compose run --rm digest python trends.py            # за 3 месяца -> trends_report.md
docker compose run --rm digest python trends.py --months 6 # за полгода
```

Сбор за 3 месяца и десяток+ вызовов модели идут несколько минут. Готовый файл
`trends_report.md` бот присылает по команде **/trends** (админам). Модель — `TRENDS_MODEL`
в `.env` (по умолчанию как `PAPER_MODEL`/`CLAUDE_MODEL`; для синтеза уместен `opus`).

## Управление каналами через бота

Отдельный процесс `bot.py` (в Docker — сервис `bot`) слушает команды и правит
`channels.txt`, который дайджест перечитывает при следующем запуске:

- `/list` — показать текущие каналы;
- `/add @channel [ещё @channel/ссылки]` — добавить (принимает `@username`, `username`
  и ссылки `https://t.me/username`);
- `/remove @channel [...]` — удалить;
- `/help` — справка.

Он же меняет расписание публикаций на лету:

- `/times` — показать текущее расписание;
- `/set digest HH:MM` — время ежедневного дайджеста;
- `/set paper HH:MM` — время научного разбора;
- `/set weekly HH:MM` — время недельного обзора;
- `/set weekday N` — день недельного обзора (1=Пн … 7=Вс).

Времена хранятся в `schedule.conf` (общий bind-mount), планировщик перечитывает его
каждую минуту, поэтому изменения применяются почти сразу без перезапуска. Файл создаётся
при первом старте из переменных `RUN_AT`/`PAPER_AT`/`WEEKLY_AT`/`WEEKLY_DOW` (compose);
дальше источник истины — он (чтобы вернуться к env-значениям, удалите `schedule.conf`).

Команды выполняют только пользователи, чьи id указаны в `BOT_ADMINS` (через запятую).
Свой id узнать просто: напишите боту любое сообщение — он ответит `Ваш id: …`.
Проверка каналов идёт только по формату username; нечитаемый канал проявится как
`0 постов` в логах дайджеста. Локально бот запускается так же: `python bot.py`.

## Автодеплой на сервер (GitHub Actions)

Workflow `.github/workflows/deploy.yml` при каждом обновлении ветки `main`
(в том числе при слиянии PR из другой ветки) заходит на сервер по SSH и обновляет
контейнеры: `git reset --hard origin/main` + `docker-compose up -d --build`.

Что нужно один раз настроить:

1. На сервере уже должен лежать развёрнутый проект (`git clone`, заполненные `.env`
   и `channels.txt`, рабочие сессия Telethon и OAuth-токен). Деплой только обновляет код.
2. Создайте SSH-ключ для деплоя и положите публичную часть на сервер:
   ```bash
   ssh-keygen -t ed25519 -f deploy_key -N ""     # на своей машине
   ssh-copy-id -i deploy_key.pub USER@SERVER      # или вручную в ~/.ssh/authorized_keys
   ```
3. В GitHub: **Settings → Secrets and variables → Actions → New repository secret** — добавьте:
   - `DEPLOY_HOST` — IP или домен сервера;
   - `DEPLOY_USER` — пользователь SSH (например `root`);
   - `DEPLOY_PATH` — путь к проекту на сервере (например `/root/ai-digest-project/ai-digest`);
   - `DEPLOY_SSH_KEY` — приватный ключ целиком (содержимое файла `deploy_key`);
   - `DEPLOY_PORT` — порт SSH, если не 22 (необязательно).

После этого любой merge в `main` автоматически выкатывается на сервер. Деплой можно
запустить и вручную: вкладка **Actions → Deploy to server → Run workflow**.
`channels.txt` из гита выведен (его правит бот), поэтому деплой его не затирает.

## Почему не гонять сам дайджест в GitHub Actions

Технически можно (через `claude setup-token` и секреты), но сессия Telethon и
OAuth-токен подписки — чувствительные данные, которые в Actions придётся хранить
в секретах и постоянно продлевать. Для личной задачи «раз в день» дешёвый VPS,
домашний мини-ПК или всегда включённый рабочий сервер надёжнее и проще.

## Траблшутинг

| Симптом | Причина / решение |
|---|---|
| `claude -p` просит API-ключ | Не выполнен вход по подписке: запустите `claude` интерактивно |
| Списываются деньги с API | В окружении есть `ANTHROPIC_API_KEY` — удалите |
| `FloodWaitError` от Telethon | Слишком частые запросы: увеличьте `asyncio.sleep` |
| Бот не постит | Бот не админ канала, либо неверный `TG_TARGET_CHANNEL` |
| Дайджест обрезан | Telegram-лимит 4096 символов — скрипт сам режет на части |
